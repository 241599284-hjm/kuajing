# 后台列表页与配置页形态规范

## 列表型业务

商品、分类、订单、客户、退款、评价、商品导入任务、死信任务、媒体对账任务、审计日志等可由业务唯一 ID 标识的数据，统一采用：

- 单条数据固定一行，只展示摘要字段，超长文本省略。
- 顶部提供新增或批量操作按钮；新增内容使用页面唯一弹窗。
- 右侧固定操作列提供详情、修改等明确按钮，点击行空白不触发弹窗。
- 详情和修改均按业务唯一 ID 独立请求完整数据；关闭弹窗清空状态，再次打开重新请求。
- 写操作必须有加载锁、错误状态和二次确认；成功后关闭弹窗并刷新当前分页。
- 弹窗与列表 DOM 平级，支持关闭按钮、遮罩、ESC、移动安全边距和底层滚动锁。

## 配置型业务

PayPal 密钥、跨境规则、SMTP、SSL/CDN、站点基础配置等全局单例配置保留配置表单，不强行改成列表。配置页必须：

- 按职责拆分区块，不循环渲染重复表单。
- 保存危险配置前二次确认。
- 涉及外部接口时提供真实测试连通性按钮。
- Secret 永不回显，失败不得伪报成功。

配置页内部若包含模板、账号、日志、资质等多记录子资源，子资源仍按列表型业务规范处理。
