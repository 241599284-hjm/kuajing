# 工艺品跨境管理后台设计与功能规范

## 1. 选定方向

- 视觉基准：静瓷后台（Quiet Porcelain）。
- 适用业务：茶具、工艺品跨境独立站的商品、订单、PayPal、物流、客户和系统运营。
- 识别锚点：青瓷绿成功状态、灰蓝主操作、极细分组线、高密度留白表格。
- 禁止：紫色渐变、米色主导、营销页 Hero、嵌套 Card、过度圆角、装饰光斑、页面贴边、纯文字状态。

## 2. 全局 Layout

- Header 固定 48px；桌面 Sidebar 240px，可收缩至 64px；Main 独立滚动。
- 平板默认图标侧栏；移动端 Sidebar 为 240px 抽屉，Main 宽度不得超过视口。
- Main 使用 `pt-6 pb-12 px-4 sm:px-6`，页面模块统一 `space-y-6`。
- Header 包含菜单、后台标识、全局搜索、币种、通知和用户菜单。
- 用户菜单必须包含账号设置、修改密码和退出登录。

## 3. 设计 Token

- 背景 `#f9fafb`，表面 `#ffffff`，边框 `#e5e7eb`。
- 正文 `#1f2937`，次要 `#6b7280`，占位 `#9ca3af`。
- 主操作灰蓝 `#55718b`；成功 `#15805d`；待处理 `#b65f12`；风险 `#cf3f3f`。
- Card、Modal 最大圆角 8px；阴影仅用于独立模块和弹窗。
- 页面标题 20px/600，Card 标题 16px/500，表单/表格 14px，备注 12px。

## 4. 组件规则

- 新页面只使用 `components/ui` 的 Button、Card、Badge、Input、Textarea、Table、Dialog 等 shadcn 风格组件。
- 图标统一使用 Lucide；图标按钮必须有可访问名称。
- 状态必须使用语义 Badge：支付完成绿色、待处理橙色、退款/取消红色。
- 表格固定表头、行 hover、横向容器滚动；表格最小宽度不得撑开页面。
- 表单按“标签、输入、提示”排列；桌面可双栏，移动端必须单栏。

## 5. 页面与功能矩阵

- 仪表盘：PayPal 订单、销售额、待发货、退款统计，7 日趋势、最新订单、库存预警。
- 订单：列表、筛选、分页、导出、物流、详情、捕获、取消、部分/全额退款。
- PayPal：订单详情、退款记录、Webhook 日志、Sandbox、Live、Webhook 订阅。
- 商品：列表、新增/编辑、分类、媒体、跨境字段、库存预警和商品导入。
- 客户：买家列表和账号状态，只展示真实客户接口数据。
- 物流：渠道、模板、账号额度、轨迹和邮件通知。
- 系统：网站基础配置、邮件、折扣、评价、DLQ、媒体对账、运维和审计。
- 首页运营：首页模块排序、显隐、复制、删除、双语内容、媒体、PC/手机预览、草稿和发布；邮件订阅必须提供真实列表、邮箱搜索、状态筛选、分页、CSV 导出、退订/恢复二次确认和操作审计。
- 用户未逐项描述的后台页面，必须复用本规范补齐标题、状态、加载、空数据、错误、分页、确认和审计入口。

## 6. 安全与确认

- 删除、退款、捕获、取消、发布、作废、生产密钥、Webhook 订阅和影响线上流量的配置必须二次确认。
- 二次确认必须说明对象、金额/环境和影响，不允许只显示“确定吗”。
- Secret 只提交到服务端加密存储，前端永不回显原文，不写日志。
- 未认证返回 401；无权限返回 403；退款只允许 owner/finance。

## 7. 接口连通性

- 每个 Provider、密钥、Webhook、物流、邮件、对象存储、CDN、分析配置必须提供“测试连通性”。
- 按钮必须调用真实后端检测端点，展示测试中、成功、失败和 correlation ID；禁止本地延时模拟成功。
- 保存与测试分开；测试通过不等于已保存，保存成功也不等于外部 Provider 可用。
- Sandbox 与 Live 凭据隔离存储和检测，不允许自动复制或混用。

## 8. PayPal 边界

- Orders v2 负责创建、查询、更新、授权和捕获；退款使用 Payments v2 Capture Refund；物流轨迹使用 PayPal Tracking API，不冒充 Orders v2 能力。
- Webhook 使用官方事件名，并在验签、durable inbox、金额/币种核对后更新本地状态。
- 有效 Sandbox 凭据、真实签名事件和真实资金退款完成前，界面必须显示“未验证”，不能显示“已接通”。
